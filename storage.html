Client-Side Data Storage

Sometimes you may want for your application to be able to remember something. For example for calculations or for passwords. Since we cannot use JavaScript variables because those are thrown away every time a page is closed, we must find a way to solve that problem.

We could set up a server to manage the data making your app a server-side persistant application. But sometimes it is ok to keep data in the browser, effectively writting less code and obtaining the same results. This approach will make your application client-side persistant.

For Client-Side storage we could set up Cookies which will cover briefly OR we could use the two mechanisms provided by the Web Storage API to store key/value pairs. We are talking about localStorage and sessionStorage. Both are properties of the Window object.

localStorage

Invoking localStorage will create an instance of the Storage object through which data can be set, retrieved or removed. The localStorage object has no expiration set and stores ALL data as String data in a way that survives page reloads.

sessionsStorage

As opposed to localStorage, data stored as sessionStorage is cleared when the page sessions ends. This could be useful for storing calculation values.
